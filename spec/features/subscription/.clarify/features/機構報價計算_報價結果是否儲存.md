# 釐清問題

報價計算的結果是否需要儲存？還是每次重新計算？

# 定位

Feature：`spec/features/subscription/機構報價計算.feature`
ERM：`spec/erm-subscription.dbml` 中未定義報價記錄表

# 多選題

| 選項  | 描述                             |
| ----- | -------------------------------- |
| A     | 不儲存，僅供計算參考             |
| B     | 儲存為草稿，可多次調整           |
| C     | 儲存為正式報價，可查詢歷史記錄   |
| D     | 報價即合約草稿，轉換為合約時儲存 |
| Short | 提供其他簡短答案（<=5 字）       |

# 影響範圍

- 影響是否需要新增 `quotations` 表
- 影響報價到簽約的流程設計
- 影響報價歷史的可追溯性

# 優先級

Low

理由：

- Feature 描述為「使用報價計算機」
- 暗示為工具性質，可能不需儲存
- 不影響核心簽約流程
- 可後續根據需求擴充

---

# 解決記錄

- **回答**：產生報價連結，供查詢、分享、業務聯絡。儲存前需填寫聯絡方式（姓名、電話、mail、方便連絡時間）。
- **更新的規格檔**：spec/features/subscription/機構報價計算.feature
- **變更內容**：新增「報價結果儲存與分享」規則，使用者點選「儲存報價」後需填寫聯絡方式，完成後產生唯一報價連結供查詢、分享與業務聯絡。
