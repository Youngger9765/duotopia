# 釐清問題

點數加購時，是否允許同時購買多個點數包（如：同時買 2 個 10k 包）？

# 定位

Feature：`spec/features/subscription/點數加購.feature` - 規則「加購 10k 點數包」與「加購 50k 點數包」

# 多選題

| 選項  | 描述                           |
| ----- | ------------------------------ |
| A     | 允許，可在同一筆訂單購買多個   |
| B     | 不允許，一次只能購買一個點數包 |
| C     | 允許，但需分別建立多筆加購記錄 |
| D     | 依付款方式不同而異             |
| Short | 提供其他簡短答案（<=5 字）     |

# 影響範圍

- 影響 `points_purchases` 表的設計（是否需要數量欄位）
- 影響加購功能的 UI 與流程
- 影響點數發放邏輯

# 優先級

Medium

理由：

- Feature 每個範例都是單一點數包
- 未定義是否支援批量購買
- 影響資料模型設計（是否需要 quantity 欄位）
- 影響使用者體驗
