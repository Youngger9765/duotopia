# 釐清問題

付款失敗時，是否允許重試？重試流程如何設計？

# 定位

功能模型：`spec/features/subscription/機構簽約與付款.feature`

# 多選題

| 選項  | 描述                                                  |
| ----- | ----------------------------------------------------- |
| A     | 付款失敗不可重試，需重新建立付款記錄                  |
| B     | 付款失敗可重試，限次數（如 3 次），重試失敗需人工處理 |
| C     | 付款失敗可無限重試，直到成功                          |
| D     | 付款失敗後可切換付款方式重試                          |
| Short | 提供其他簡短答案（<=5 字）                            |

# 影響範圍

- 影響付款失敗處理邏輯
- 影響訂閱開通流程
- 影響財務記錄與錯誤追蹤
- 需新增付款失敗重試的 Feature Rules

# 優先級

High

理由：

- 付款失敗是常見場景，需明確定義重試規則
- 影響訂閱開通與合約生效流程
- 影響用戶體驗與財務安全

---

# 解決記錄

- **回答**：付款失敗由業務人員手動重新開立付款方式
- **更新的規格檔**：spec/features/subscription/機構簽約與付款.feature
- **變更內容**：新增「付款失敗處理」規則，明確定義由業務人員手動重開付款方式，系統記錄新付款資訊。
