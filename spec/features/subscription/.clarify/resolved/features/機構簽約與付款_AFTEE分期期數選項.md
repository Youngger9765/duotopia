# 釐清問題

AFTEE 分期的期數是否固定？還是可以自選？

# 定位

Feature：`spec/features/subscription/機構簽約與付款.feature` - 規則「處理付款 - AFTEE 分期」

# 多選題

| 選項  | 描述                                          |
| ----- | --------------------------------------------- |
| A     | 固定：一年約只能 12 期，兩年約只能 24 期      |
| B     | 可選：一年約可選 6/12 期，兩年約可選 12/24 期 |
| C     | 完全自選：任意選擇 1-24 期                    |
| D     | 依合約金額不同有不同選項                      |
| Short | 提供其他簡短答案（<=5 字）                    |

# 影響範圍

- 影響 `payments.installment_periods` 的驗證規則
- 影響付款方式選擇 UI 的期數選項
- 影響 AFTEE 整合的參數設定

# 優先級

Medium

理由：

- Feature 範例顯示「一年約分 12 期，兩年約分 24 期」
- 未明確說明是唯一選項還是範例之一
- 需確認 AFTEE 的實際支援範圍
- 影響 UI 設計

---

# 解決方案（決策 #6）

**決策日期**: 2026-01-09

**答案**: 架構調整 - 改為手動記錄

**說明**:

- AFTEE 分期為線下處理
- 專案人員記錄實際分期期數（installment_periods）
- 不需限制選項，由業務與機構協商

**相關更新**:

- ✅ payments.installment_periods 保持為 int (可選)
- ✅ 新建「專案人員開立機構訂閱.feature」- 包含 AFTEE 分期記錄場景
