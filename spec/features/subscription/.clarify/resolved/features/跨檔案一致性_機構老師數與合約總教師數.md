# 釐清問題

報價計算中「機構有 N 位英文老師」與簽約時「總教師數」的對應關係？

# 決策

**選項**: C - 機構英文老師數是參考，總教師數可調整

## 決策說明

**報價計算機的角色**：

- 「機構需要 N 位教師授權」是報價計算的**輸入參數**
- 可以與「機構實際英文老師數」相同，也可以不同
- 報價計算機只負責計算：「如果需要 N 位授權，費用是多少」

**實際簽約時**：

- 機構根據報價與預算，決定實際購買的 `total_teachers`
- 業務人員根據商務談判，決定 `purchased_teachers` 和 `bonus_teachers` 的拆分
- `total_teachers` 可能大於、等於、或小於機構實際英文老師數

**範例情境**：

1. 機構有 20 位英文老師，但預算有限，可能只購買 15 位授權
2. 機構有 8 位英文老師，但預期擴編，可能購買 12 位授權
3. 機構有 10 位英文老師，剛好購買 10 位授權

## 推斷依據

- 報價計算是彈性工具，提供不同情境的費用試算
- 實際購買數由機構預算與規劃決定
- 「機構有幾位老師」僅作為規劃參考

# 解決記錄

- **回答**：C - 報價計算機僅用 purchased_teachers，贈送教師數量由業務人工確認，合約時才納入 total_teacher_licenses
- **更新的規格檔**：spec/features/subscription/機構報價計算.feature、spec/features/subscription/機構簽約與付款.feature
- **變更內容**：報價計算機不含贈送教師數量，僅以使用者輸入的教師授權數（purchased_teachers）計算，合約建立時才納入 total_teacher_licenses（含贈送教師數量）

- 「機構需要 N 位教師授權」（而非「機構有 N 位英文老師」）
- 強調這是報價計算的輸入參數，不代表實際老師數

### Feature 用詞建議

**一致的表述**：

- 報價計算 Feature：「機構需要 N 位教師授權」
- 簽約 Feature：「機構有 N 位英文老師」+「總教師數為 M 位」

這樣可清楚區分：

- 報價階段：靈活試算不同授權數的費用
- 簽約階段：確定實際開通的授權數
