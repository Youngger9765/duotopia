# 釐清問題

合約中教師月費的定價規則是什麼？是否為固定值？

# 定位

ERM：`spec/erm-subscription.dbml` - Table `contracts` - 欄位 `teacher_monthly_price`

# 多選題

| 選項  | 描述                                                         |
| ----- | ------------------------------------------------------------ |
| A     | 固定值：一年約 10.0 USD，兩年約 9.0 USD                      |
| B     | 依購買教師數量分級定價（如：1-10 位 10 USD，11-20 位 9 USD） |
| C     | 可協商的浮動價格，每個合約可不同                             |
| D     | 有折扣機制（如：學生數超過 100 有折扣）                      |
| Short | 提供其他簡短答案（<=5 字）                                   |

# 影響範圍

- 影響 `contracts.teacher_monthly_price` 欄位的約束條件
- 影響報價計算邏輯
- 影響合約建立時的欄位驗證規則

# 優先級

Medium

理由：

- DBML 的 Note 提到「一年約 10.0，兩年約 9.0」
- Feature 文件也使用這些固定值
- 但欄位類型為 decimal，暗示可能有浮動空間
- 需確認是否完全固定，或未來可能調整

---

# 解決方案（決策 #9）

**決策日期**: 2026-01-09

**答案**: 系統級標準價格（可由管理員調整，業務無權修改）

**說明**:

- 當前標準價格：
  - 一年約：10.0 USD/月
  - 兩年約：9.0 USD/月
- 價格由系統管理員統一設定（未來可漲價）
- 業務人員簽約時套用當時的標準價格
- 業務人員無權針對個別合約調整價格

**建議實作**:

- 新增價格設定表 `pricing_config` 或類似機制
- 合約記錄簽約時的價格快照（teacher_monthly_price）
- 價格歷史記錄供追溯

**影響**:

- 報價計算機使用當前標準價格
- 合約建立時自動套用標準價格
- 未來漲價不影響既有合約

**信心評分**: High
