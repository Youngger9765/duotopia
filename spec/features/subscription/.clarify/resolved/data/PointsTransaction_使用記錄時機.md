# 決策紀錄

## 決策日期
2026-01-10

## 問題
點數使用 (usage) 的記錄時機是什麼？每句計費還是批次計費？

## 決策
A-D 皆需支援：
- A. 每次朗讀提交時即時扣點並記錄
- B. 每日批次結算當天使用量
- C. 每週批次結算
- D. 每月批次結算

## 補充
- 每種時機皆需能產生 usage 交易紀錄，並必須繫結實際使用者（如屬批次結算，則每筆紀錄仍需標註對應使用者與結算區間）。
- 實務上，A（即時）為預設主流程，B-D 為特殊合約或大客戶需求時的批次結算選項。

## 影響範圍
- 交易記錄建立頻率與資料量需依據結算時機調整
- 剩餘點數即時性依據結算模式而異
- 需考量效能、資料庫設計與批次結算流程

## 信心評分
High

## 決策重點
- 系統需同時支援即時與批次 usage 記錄
- 每筆 usage 交易必須繫結實際使用者，便於追蹤與稽核
- 批次結算時需明確標註結算區間與對應明細
