# 釐清問題

合約中點數單價是否可變？還是永遠固定為 0.0015 TWD？

# 定位

ERM：`spec/erm-subscription.dbml` - Table `contracts` - 欄位 `points_unit_price`

# 多選題

| 選項  | 描述                                     |
| ----- | ---------------------------------------- |
| A     | 永遠固定為 0.0015 TWD，所有合約相同      |
| B     | 可依合約類型不同（如：兩年約有優惠單價） |
| C     | 可依購買點數量不同（如：大量購買有折扣） |
| D     | 可依簽約時間不同（如：早鳥優惠）         |
| Short | 提供其他簡短答案（<=5 字）               |

# 影響範圍

- 影響 `contracts.points_unit_price` 欄位的預設值與約束
- 影響報價計算功能的邏輯
- 影響未來價格調整的靈活性

# 優先級

Low

理由：

- 目前所有 Feature 文件都使用 0.0015 這個固定值
- DBML 已設定 default: 0.0015
- 即使未來可變，也不影響目前的核心功能實作
- 可先實作為固定值，未來再擴充

---

# 解決方案（決策 #10）

**決策日期**: 2026-01-09

**答案**: 系統級標準價格（訂約時最優惠，未來可能漲價）

**說明**:

- 當前標準價格：0.0015 TWD/點
- 訂立合約當下為最優惠價格
- 簽約時自動套用當時標準價格並記錄快照
- 業務人員無權調整價格
- 未來可能漲價（由管理員統一調整）
- 漲價不影響既有合約（已記錄快照）

**與決策 #9 的一致性**:

- 教師月費與點數單價採用相同定價機制
- 都是系統級標準價格
- 都可由管理員未來調整
- 都在簽約時記錄快照

**建議實作**:

- 使用相同的價格設定表機制（pricing_config）
- 合約記錄簽約時的價格快照（points_unit_price）
- 價格歷史記錄供追溯

**影響**:

- 報價計算機使用當前標準價格
- 合約建立時自動套用標準價格
- 未來漲價不影響既有合約

**信心評分**: High
