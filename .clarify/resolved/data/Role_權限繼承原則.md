# 釐清問題

角色層級的權限繼承原則為何？

# 定位

核心規則：角色層級關係中的權限繼承邏輯。

# 多選題

| 選項 | 描述                                                                                              |
| ---- | ------------------------------------------------------------------------------------------------- |
| A    | 上層角色自動繼承下層角色的所有權限（如：org_owner 自動擁有 teacher 的所有權限）                   |
| B    | 不同層級角色權限完全獨立，無繼承關係                                                              |
| C    | 僅在同一體系內繼承（如：機構內 org_owner > org_admin > school_principal，但與個人教師無繼承關係） |
| D    | 依據具體情境動態判斷（某些操作繼承，某些不繼承）                                                  |

# 影響範圍

- 權限驗證邏輯的複雜度
- 角色權限矩陣的設計
- Casbin 權限策略的繼承規則
- 跨層級操作的權限判斷
- 權限測試案例的覆蓋範圍

# 優先級

High

---

# 解決記錄

- **回答**：A - 上層角色自動繼承下層角色的所有權限（適用於機構後台 RBAC 體系）
- **更新的規格檔**：docs/specs/user-roles-and-permissions.md
- **變更內容**：在「核心規則 > 角色層級關係」章節記錄機構後台的權限繼承原則
