# 釐清問題

用戶擁有多角色時的行為邏輯為何？

# 定位

核心規則：跨類型角色的處理機制。

# 多選題

| 選項 | 描述                                           |
| ---- | ---------------------------------------------- |
| A    | 用戶需手動切換角色，每次操作僅使用當前角色權限 |
| B    | 權限自動合併（用戶擁有所有角色的權限聯集）     |
| C    | 依據操作情境自動選擇最合適的角色               |
| D    | 設定預設角色，但可臨時切換至其他角色           |

# 影響範圍

- 用戶登入後的預設角色選擇邏輯
- 角色切換功能的 UI/UX 設計
- 權限驗證時的角色判斷邏輯
- 多角色用戶的資料訪問範圍
- 操作日誌記錄的角色標記

# 優先級

High
