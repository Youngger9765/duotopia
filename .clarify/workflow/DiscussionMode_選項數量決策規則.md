# 釐清問題

當提供多選題時，應該如何決定給出 2 個還是 5 個選項？選項數量的決策規則是什麼？

# 定位

discussion.md：建議模式流程的「第三步：提出選項」，目前規定「每次確認都應該給我 2-5 個選項」，但未說明如何決定具體數量。

# 多選題

| 選項 | 描述                                                                         |
| ---- | ---------------------------------------------------------------------------- |
| A    | 固定給 3-4 個選項，保持一致性                                                |
| B    | 根據問題複雜度決定：簡單問題 2-3 個，複雜問題 4-5 個                         |
| C    | 根據方案多樣性決定：若有明顯不同的實作方向則多給選項，若差異小則少給選項     |
| D    | 優先給 2-3 個核心選項，若用戶表示「還有其他選項嗎？」再補充                  |
| E    | 根據決策影響決定：高影響決策（架構、資料模型）給 4-5 個，低影響決策給 2-3 個 |

# 影響範圍

- 用戶決策效率（選項太多會造成選擇困難）
- AI 分析深度（準備更多選項需要更多分析時間）
- discussion.md 的「建議模式流程 > 第三步：提出選項」章節內容
- 與 clarify-and-translation.md 中多選題格式的一致性

# 解決記錄

- **回答**：D - 優先給 2-3 個核心選項，若用戶表示「還有其他選項嗎？」再補充

- **說明**：

  **基本原則：先給核心選項**

  - 優先提供 2-3 個最具代表性、差異最明顯的核心選項
  - 避免一開始就給出過多選項造成選擇困難
  - 確保每個核心選項都有明確的適用場景和優缺點

  **彈性擴展機制**

  - 若用戶表示「還有其他選項嗎？」或「有沒有別的方案？」
  - AI 可補充 1-2 個額外選項（總計不超過 5 個）
  - 補充選項應說明為何一開始未列入核心選項（例如：適用場景較窄、實作複雜度較高等）

  **選項品質優於數量**

  - 每個選項都應有實質差異（不同技術方向、不同權衡取捨）
  - 避免為了湊數量而提出相似的選項
  - 若只有 2 個明顯不同的方案，就只給 2 個，不強行擴充
