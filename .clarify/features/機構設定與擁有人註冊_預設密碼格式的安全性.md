# 釐清問題

機構設定與擁有人註冊.feature 中，未註冊擁有人的預設密碼為「Aa + 統編 + @」，此格式是否符合安全要求？是否有密碼強度驗證？是否強制首次登入修改？

# 定位

Feature: 機構設定與擁有人註冊.feature
規則: 新增機構並設定擁有人 - 擁有人尚未註冊
場景: 為未註冊的機構擁有人發送認證信
Then: "預設密碼應為 'Aa87654321@'"
註解: "# 格式: Aa + 統編 + @"

# 多選題

| 選項 | 描述 |
|------|------|
| A | 強制首次登入修改密碼 |
| B | 統編是公開資訊，建議改用隨機密碼 |
| C | 保持此格式，但加入密碼強度驗證 |
| D | 使用臨時 token 取代預設密碼 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 影響安全性策略
- 影響帳號註冊流程
- 影響密碼政策設計
- 需評估統編作為密碼基礎的風險

# 優先級

High

- 安全性問題
- 統編為公開資訊，密碼可預測
- 建議改用隨機密碼或 token
