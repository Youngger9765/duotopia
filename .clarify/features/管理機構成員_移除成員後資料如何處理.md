# 釐清問題

管理機構成員.feature 說明移除成員時「teacher_organizations 表中對應記錄的 is_active 更新為 false」，但沒有說明：
- 該教師在學校的角色是否同時移除？
- 該教師建立的班級/教材是否保留？
- 該教師的歷史操作記錄如何處理？

# 定位

Feature: 管理機構成員.feature
Rule: 機構負責人可以移除機構管理人
Example: 成功移除機構管理人

# 多選題

| 選項 | 描述 |
|------|------|
| A | 僅移除機構角色，學校角色保留 |
| B | CASCADE 移除所有相關角色與資料 |
| C | 移除角色但保留歷史資料 |
| D | 需確認後才移除（防止誤刪） |
| E | 分階段移除（先停用，後刪除） |

# 影響範圍

- 影響資料完整性設計
- 影響外鍵 CASCADE 設定
- 影響成員移除的 UI 流程
- 影響歷史資料保留策略
- 需補充 Example 說明完整行為
