# 釐清問題

管理機構成員.feature 的「編輯成員角色」Rule 中，Example 顯示「機構負責人將機構管理人提升為機構負責人」，但沒有說明原本的 org_owner 如何處理（自動降級？拒絕操作？）

# 定位

Feature: 管理機構成員.feature
Rule: 編輯成員角色
Example: 機構負責人將機構管理人提升為機構負責人

# 多選題

| 選項 | 描述 |
|------|------|
| A | 自動將原 org_owner 降級為 org_admin |
| B | 拒絕操作，需先移除原 org_owner |
| C | 支援轉移流程（兩階段確認） |
| D | 允許暫時兩個 org_owner，手動處理 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 影響角色編輯邏輯
- 影響「一個機構一個 org_owner」規則的執行
- 影響 UI 流程設計
- 需補充 Example 說明完整轉移流程

# 優先級

High

- 與核心規則「一個機構僅能有一個機構負責人」衝突
- 需明確定義轉移邏輯
