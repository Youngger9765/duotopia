# 釐清問題

個人教師與機構教師身分切換.feature 說明「班級應標示所屬：『個人』或『機構-台北分校』」，但沒有說明：
- 標示來源是資料庫欄位還是運算結果？
- 如何判斷班級是個人還是機構？
- classroom 表是否需要新增 type 或 source 欄位？

# 定位

Feature: 個人教師與機構教師身分切換.feature
規則: 在個人教師後台查看資料
場景: 被指派為班級老師的機構教師查看後台
Then: "班級應標示所屬: '個人' 或 '機構-台北分校'"

ERM: classroom_schools 表（判斷依據？）

# 多選題

| 選項 | 描述 |
|------|------|
| A | 查詢 classroom_schools 表，有記錄即為機構班級 |
| B | classroom 表新增 source 欄位（individual/organization） |
| C | 前端運算邏輯（根據關聯查詢） |
| D | classroom 表新增 organization_id 欄位 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 影響資料模型設計（是否需新增欄位）
- 影響查詢邏輯與效能
- 影響前端顯示邏輯
- 需在 DBML 或 Feature 中明確說明

# 優先級

Medium

- 影響實作設計
