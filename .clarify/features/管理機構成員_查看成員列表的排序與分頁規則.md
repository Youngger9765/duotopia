# 釐清問題

管理機構成員.feature 中「查看機構成員列表」的 Example 返回成員列表，但沒有說明：
- 列表排序規則（按角色？加入時間？姓名？）
- 是否支援分頁（機構成員可能很多）
- 是否支援搜尋與篩選

# 定位

Feature: 管理機構成員.feature
Rule: 查看機構成員列表
Example: 機構負責人查看所有成員

# 多選題

| 選項 | 描述 |
|------|------|
| A | 按角色優先級排序（org_owner > org_admin > teacher） |
| B | 按加入時間排序（created_at） |
| C | 支援分頁與搜尋 |
| D | 此為 API 實作細節，Feature 不明確規範 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

- 影響 API 設計（排序、分頁參數）
- 影響前端列表實作
- 不影響核心業務邏輯

# 優先級

Low

- API 設計細節
- 不影響核心功能
- 可在實作階段決定
