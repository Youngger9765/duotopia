# 釐清問題

teacher_schools.permissions 說明可「覆蓋預設角色權限」，但沒有說明覆蓋的優先級、合併邏輯或驗證規則是什麼？

# 定位

ERM：teacher_schools 表，permissions 欄位
Note: "自訂權限設定（JSON 格式），可覆蓋預設角色權限"

# 多選題

| 選項 | 描述 |
|------|------|
| A | 完全覆蓋（permissions 優先於 roles） |
| B | 合併策略（取聯集或交集） |
| C | 僅允許限縮權限，不可擴大 |
| D | 僅允許擴大權限，不可限縮 |
| E | 需明確定義覆蓋規則 |

# 影響範圍

- 影響權限檢查邏輯（Casbin 策略）
- 影響 API 設計（權限管理端點）
- 影響前端權限設定介面
- 影響測試設計（權限覆蓋 Example）

# 優先級

High

- 影響核心權限系統設計
- 影響安全性
- 需在規格中明確定義
