## Key Takeaways

### ğŸ”’ Lock Mechanism
- `SELECT FOR UPDATE` on Organization prevents concurrent modifications to the org itself
- But it doesn't prevent concurrent inserts into related tables (TeacherOrganization)

### âœ… Pre-Check (Optimization)
- Fails fast for requests that would obviously exceed limit
- Reduces unnecessary inserts and rollbacks

### ğŸ” Post-Check (Security)
- The critical protection against race conditions
- Verifies the FINAL state after insert but before commit
- Sees all changes from current transaction + committed changes from other transactions

### âŒ Rollback
- Only triggered when post-check fails
- Cleanly undoes the insert without side effects
- Returns clear error message to client

### ğŸ’¡ Why `flush()` is Essential
```python
db.flush()   # Writes to DB but transaction still open
             # Post-check can see the inserted row
             # Can still rollback if needed

db.commit()  # Transaction closes
             # No chance to verify or rollback
             # Race condition window
```
